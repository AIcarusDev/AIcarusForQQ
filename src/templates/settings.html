📧 掉线告警（SMTP 邮件）

    

      

        NapCat 作为第三方插件存在被风控/掉线的风险。启用后会在
        WebSocket 断开 或 心跳超时 时向指定邮箱发送提醒。
        SMTP 凭据保存在 .env 文件中（密码字段以 *
        掩码显示，不修改则保留原值）。
      

      

        启用掉线告警
        
      

      

        

          
          
          连续多少秒未收到 NapCat 心跳即视为掉线（建议 90~180）
        

        

          
          
          两次同类告警最小间隔，防骚扰
        

      

      

        
        
      


      

        

          SMTP 服务器配置
        

        

          

            
            
          

          

            
            
          

        

        

          使用 SSL（端口 465 选 SSL；587 用 STARTTLS 请取消勾选）
          
        

        

          

            
            
            SMTP 服务器认证用的账号
          

          

            
            
            已设置时显示 *，留空或保持掩码不修改
          

        

        

          
          
          仅在需要"代发"时填写，与 USER 不一致时部分服务商会拒绝或标垃圾邮件
        

        

          
          
        

        

          
          点击前请先「保存并应用」让最新 SMTP 配置生效
        

      

    

  


  
  

    
🚦 NapCat 自动重启（断线后自动拉起）

    

      

        启用后，当收到 bot_offline 通知、WebSocket 断开 或
        心跳超时 时，会先尝试 关闭已有 NapCat 进程，再执行
        command 启动 NapCat。恢复窗口期
        内若仍未连上，会扫描配置的二维码路径并作为附件邮件发送给你扫码续登。

        多 NapCat 安全：默认按 cwd 的父目录前缀过滤进程，
        不会误杀其他位置的 NapCat 实例（除非显式开启「按映像名强杀」）。
      

      

        启用 NapCat 自动重启
        
      

      

        
        
        推荐指向 napcat.quick.bat（需配合环境变量
          ACCOUNT / NAPCAT_QUICK_PASSWORD_MD5
          实现免扫码登录）
      

      

        

          
          
        

        

          
          
        

      

      

        
        
        若提供则优先执行；注意脚本必须是 非交互
          的（不能含 set /p 之类等待输入）
      

      

        

          
          
        

        

          
          
          只有 ExecutablePath 以该前缀开头的进程才会被杀
        

      

      

        按映像名强杀（force_kill_by_image_name，慎用，会杀掉本机所有同名 NapCat）
        
      

      

        

          
          
        

        

          
          
        

      

      

        

          
          
        

        

          
          
          窗口内仍未连上则视为需要扫码，会发送二维码邮件
        

      

      

        
        
        支持绝对路径或相对路径；相对路径会在 cwd 与
          cwd 的父目录中各搜一次
